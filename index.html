Setup

Live in four steps

Static scripts and Python—no SaaS, no mandatory cloud. You own every file on your network.

  1. Run models once

    Enable a local OpenAI-compatible API on your home server—a container runtime with a model runner works great.

  2. Sync the desk

    One command refreshes knowledge, RAG exports, runbooks, and a session handoff template.

  3. Connect each client

    Set your editor's base URL to the server IP. Add the LEMdesk MCP server for local doc search.

  4. Move rooms

    Update handoff, open the same repo, same settings, same local model—pick up where you left off.

FAQ

Common questions

Is LEMdesk a cloud service?

No. Everything runs on your machines. LEMdev provides scripts and templates you run yourself.

Does it replace Cursor or Claude?

No. It wires your existing tools to your local model and your local knowledge.

Is it affiliated with Docker or any AI vendor?

No. LEMdesk is an independent project from LEMdev. You choose which third-party tools to use.

What does it cost?

The kit is free. You may pay for hardware, electricity, and your domain—never per-seat fees from us.

Is chat history synced in Cursor?

Not natively. LEMdesk adds handoff files, optional shared browser chat, and config sync—not full Cursor cloud parity.
