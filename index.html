LEMdesk

LEMdesk — same desk on any room. Free LAN kit for local models, agent sync, knowledge search, and session handoff.

One model URL

Point every desktop at a single OpenAI-compatible endpoint on your LAN (e.g. :12434/engines/v1). No per-room model installs.

Auto knowledge

Refresh a local search corpus in seconds. MCP tools in Cursor query your docs—not the cloud.

Session handoff

Write a handoff file before you leave a desk. Open it on the next machine and continue the same thread.

Agent sync

Paths and guides for syncing skills, rules, and MCP configs across tools on your machines.

Optional shared chat

Run a browser UI on your model server so chat history lives on the network, not one laptop.

Local supervisor

Multi-agent YAML + RAG over your own markdown corpus—your stack, your machine.

Model server → LAN → Cursor / Claude → LEMdesk sync → Next room

How it works

Static scripts and Python—no SaaS, no mandatory cloud. You own the files on your network.

  1. Run models once

    Enable a local OpenAI-compatible API on your home server (many people use a desktop container runtime with a model runner).

  2. Sync the desk

    One command refreshes knowledge, RAG exports, runbooks, and a session handoff template.

  3. Connect each client

    Set your editor’s base URL to the server IP. Add the LEMdesk MCP server for local doc search.

  4. Move rooms

    Update handoff, open the same repo, same settings, same local model—pick up where you left off.

Get started

Clone the repo, run the auto pipeline, open the handoff doc.

# Clone LEMdesk (LEMdev)
git clone https://github.com/billrilea-lab/LemDesk.git
cd LemDesk

# One-shot: knowledge + RAG + briefs + handoff
python3 bot.py lemdesk-sync --fast

# Or the shell wrapper
./lemdesk/scripts/lemdesk_auto.sh --fast

# Point Cursor: Override OpenAI Base URL
# http://<server-ip>:12434/engines/v1
# API key: not-needed

Requirements: Python 3.10+, a machine that can serve local models, and clients on the same trusted LAN. See the repo README for MCP setup and optional shared chat UI.

FAQ

Is LEMdesk a cloud service?
No. Everything runs on your machines. LEMdev provides scripts and templates you run yourself.
Does it replace Cursor or Claude?
No. It wires your existing tools to your local model and your local knowledge.
Is it affiliated with Docker or any AI vendor?
No. LEMdesk is an independent project from LEMdev. Videos may mention third-party tools you choose to use.
What does it cost?
The kit is free. You may pay for hardware, electricity, and optional hosting—never per-blink fees from us.
Is chat history synced in Cursor?
Not natively. LEMdesk adds handoff files, optional shared browser chat on your server, and config sync—not full Cursor cloud parity.
